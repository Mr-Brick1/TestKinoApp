# 🎬 KinoApp — каталог фильмов с фильтрацией по жанрам

Приложение для отображения списка фильмов с фильтрацией по жанрам, детальным экраном и обработкой сетевых подключений.  
Разработано в качестве тестового задания для демонстрации навыков работы с **Retrofit**, **Coroutines**, **Picasso** и кастомными **LayoutManager**.

---

## 📱 Возможности

- **Адаптивная двухколоночная сетка** – фильмы расположены в сетке 2×N, а заголовки и жанры занимают всю ширину экрана
- **Умная фильтрация** – один клик по жанру показывает только релевантные фильмы, повторный – сбрасывает фильтр
- **Детальная карточка фильма** – локализованное и оригинальное название, год, рейтинг, описание, полный список жанров
- **Плавная загрузка изображений** – постеры подгружаются асинхронно с кешированием, округлением углов и fallback-иконкой

---

## 🛠 Используемые технологии

| Технология | Назначение |
|------------|------------|
| Kotlin | Язык разработки |
| RecyclerView + Adapter | Отображение списка с разными типами View |
| GridLayoutManager + SpanSizeLookup | Разная ширина ячеек в сетке |
| Retrofit + Gson | Загрузка данных из API (JSON) |
| Coroutines (IO + Main) | Асинхронные запросы без блокировки UI |
| Picasso + RoundedTransformationBuilder | Загрузка и округление постеров |
| ViewBinding | Безопасная работа с вёрсткой |
| Snackbar + ConnectivityManager | Обработка отсутствия интернета |

## 🧠 Принципы реализации

### 1. **Смешанный список (один RecyclerView для жанров и фильмов)**
- `Adapter` имеет три типа view: `0` – заголовок "Жанры"/"Фильмы", `1` – кнопка жанра, `2` – карточка фильма.
- `getItemViewType()` определяет тип по полю `type` в `Item`.
- `GridLayoutManager.SpanSizeLookup` задаёт разную ширину: типы 0 и 1 занимают 2 колонки, тип 2 – 1 колонку.

### 2. **Фильтрация по жанрам**
- При клике на жанр:
  - Сбрасывается выделение всех жанров.
  - Удаляются все фильмы из списка (индексы с 15).
  - Добавляются только те фильмы, у которых в списке `genres` содержится выбранный жанр.
- При повторном клике – восстанавливаются все фильмы из сохранённой копии (`SavedItems`).

### 3. **Работа с сетью и офлайн-поведение**
- Retrofit загружает данные по `https://.../films.json`.
- `MainActivity` проверяет `ConnectivityManager` – если сети нет, показывает `Snackbar` с кнопкой "ПОВТОРИТЬ" и запускает периодическую проверку каждые 3 секунды.
- После появления сети автоматически загружается `FragmentChoice`.

### 4. **Детальный экран**
- Передача данных через `Bundle` (локализованное имя, оригинальное имя, год, рейтинг, URL постера, описание, список жанров).
- Picasso с кастомным `OkHttpClient`, добавляющим заголовки `Referer` и `User-Agent` для корректной загрузки изображений.
- Отображение рейтинга с округлением до одного знака после запятой.

## 🚀 Запуск проекта

1. Клонируйте репозиторий  
2. Откройте проект в **Android Studio**  
3. Синхронизируйте Gradle (используются зависимости: Retrofit, Coroutines, Picasso, RoundedImageView)  
4. Запустите на эмуляторе или физическом устройстве с **Android 6.0+**
